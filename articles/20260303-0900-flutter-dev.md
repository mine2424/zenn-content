## はじめに

Flutterのアニメーションは、初見では「種類が多くて何を使えばいいかわからない」と感じることが多い。`AnimatedContainer`、`AnimationController`、`Tween`、`Hero`、`AnimatedBuilder`——これらが混在していて、どれがどう関係するのか整理できていない人も多いだろう。

本記事では、Flutterのアニメーションの根幹にある`AnimationController`の仕組みから出発し、Heroアニメーションまでの設計思想を一本の線でつなげて理解することを目指す。

---

## アニメーションの本質は「時間に対する値の変化」

Flutterのアニメーション系APIを理解する上でまず押さえるべきは、アニメーションの本質だ。

アニメーションとは**「時間の経過に対して、何らかの値が変化すること」**に過ぎない。ウィジェットの位置が動く、色が変わる、サイズが変わる——どれも「時間に対する値の変化」だ。

Flutterのアニメーションシステムは、この抽象概念を3つのレイヤーに分けて扱う。

1. **時間を管理する**: `AnimationController`
2. **値の変化の仕方を定義する**: `Tween` + `Curve`
3. **値の変化をUIに反映する**: `AnimatedBuilder` / `AnimatedWidget`

この3レイヤーを理解すれば、Flutterのアニメーション全体が見えてくる。

---

## AnimationController — 時間の管理者

`AnimationController`は、アニメーションの「時間軸」を管理するオブジェクトだ。`0.0`〜`1.0`の範囲で値を動かすタイマーのようなものと考えるとわかりやすい。

```dart
class _MyWidgetState extends State<MyWidget>
    with SingleTickerProviderStateMixin {

  late AnimationController _controller;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      vsync: this,           // フレームに同期させる
      duration: const Duration(milliseconds: 300),
    );
  }

  @override
  void dispose() {
    _controller.dispose();  // 必ずdisposeする
    super.dispose();
  }
}
```

`vsync: this` は重要なポイントだ。`TickerProviderStateMixin` を使うことで、アニメーションがディスプレイのリフレッシュレートに同期し、画面に表示されていないときは停止する。これによりバッテリーと処理の無駄使いを防ぐ。

`_controller.forward()` でアニメーション開始、`_controller.reverse()` で逆再生、`_controller.repeat()` でループができる。

---

## Tween — 値の変換器

`AnimationController` は `0.0`〜`1.0` しか扱わない。実際の値（例: 透明度、座標、色）への変換を担うのが `Tween` だ。

```dart
final Animation<double> _opacity = Tween<double>(
  begin: 0.0,
  end: 1.0,
).animate(_controller);

final Animation<Offset> _position = Tween<Offset>(
  begin: const Offset(0, 0.5),
  end: Offset.zero,
).animate(_controller);

final Animation<Color?> _color = ColorTween(
  begin: Colors.blue,
  end: Colors.red,
).animate(_controller);
```

`Tween.animate(controller)` で `Animation<T>` オブジェクトが生成される。これはコントローラーの値（0.0〜1.0）をbegin〜endの間で線形補間したものを返す。

---

## Curve — 変化のリズム

デフォルトは線形補間だが、`CurvedAnimation` でイージング関数を適用できる。

```dart
final Animation<double> _curved = Tween<double>(
  begin: 0.0,
  end: 1.0,
).animate(
  CurvedAnimation(
    parent: _controller,
    curve: Curves.easeOutBack,  // 行き過ぎてから戻るバウンス
  ),
);
```

`Curves.easeIn`、`Curves.easeOutCubic`、`Curves.elasticOut` など、Flutterには豊富なカーブが用意されている。アニメーションの「気持ちよさ」を決める部分だ。

---

## AnimatedBuilder でUIに反映する

値が変化しても、UIが更新されなければ意味がない。`AnimatedBuilder` を使うと、アニメーション値が変化するたびに特定のウィジェットだけを再構築できる。

```dart
@override
Widget build(BuildContext context) {
  return AnimatedBuilder(
    animation: _controller,
    builder: (context, child) {
      return Opacity(
        opacity: _opacity.value,
        child: SlideTransition(
          position: _position,
          child: child,
        ),
      );
    },
    child: const Text("アニメーション"),  // 変化しない部分はchildに分離
  );
}
```

ポイントは `child` の分離だ。アニメーション中に毎フレーム再構築されるのは`builder`内だけにして、変化しないウィジェットツリーは `child` に渡すことでパフォーマンスを保つ。

---

## 明示的 vs 暗黙的アニメーション

ここまで説明してきた `AnimationController` ベースのアニメーションは**明示的アニメーション**と呼ばれる。開発者がアニメーションを完全に制御する。

一方、`AnimatedContainer`、`AnimatedOpacity`、`AnimatedPositioned` などは**暗黙的アニメーション**だ。プロパティの値が変わると自動でアニメーションする。

```dart
// 暗黙的アニメーション — シンプルだが制御が限られる
AnimatedContainer(
  duration: const Duration(milliseconds: 300),
  width: _expanded ? 200.0 : 100.0,
  color: _expanded ? Colors.blue : Colors.red,
  curve: Curves.easeInOut,
)
```

**使い分けの原則**:
- 単純な状態変化（展開/折りたたみ、色変化）→ 暗黙的
- 複雑なタイミング制御、ループ、外部イベントとの同期 → 明示的

## Heroアニメーション — 画面遷移の橋渡し

Heroアニメーションは、異なる画面（Route）をまたいでウィジェットがアニメーションしながら移動する効果だ。

```dart
// リスト画面
Hero(
  tag: "product-image-${product.id}",  // 一意のタグ
  child: Image.network(product.imageUrl),
)

// 詳細画面
Hero(
  tag: "product-image-${product.id}",  // 同じタグ
  child: Image.network(product.imageUrl, width: double.infinity),
)
```

同じ `tag` を持つ2つのHeroウィジェットがルート遷移時に検出されると、Flutterが自動的に「出発位置から到着位置へのアニメーション」を生成する。これだけで自然な画面遷移が実現する。

**注意点**:
- タグはページ内で一意でなければならない。リストの場合はIDなどを含める
- Heroのchildはサイズが変わることが多いため、`fit: BoxFit.cover` などで対応する
- カスタムアニメーションにしたい場合は `flightShuttleBuilder` を使う

## アニメーション設計で気をつけること

### パフォーマンス

`AnimatedBuilder` の `builder` 内で重いウィジェットを毎フレーム再構築するのは避ける。不変な部分は `child` に分離し、Flutterのウィジェット再利用の仕組みを活かす。

また、`setState` を使ったアニメーション実装はウィジェットツリー全体を再構築するため非効率だ。アニメーション専用のウィジェット（`AnimatedBuilder`、`TransitionWidget`）を使うのが正しい。

### 複数のアニメーションを組み合わせる

同じコントローラーで複数のアニメーションを動かすには `Interval` が便利だ。

```dart
final Animation<double> _fade = Tween<double>(begin: 0, end: 1).animate(
  CurvedAnimation(
    parent: _controller,
    curve: const Interval(0.0, 0.5, curve: Curves.easeIn),
  ),
);

final Animation<double> _slide = Tween<double>(begin: 30, end: 0).animate(
  CurvedAnimation(
    parent: _controller,
    curve: const Interval(0.3, 1.0, curve: Curves.easeOut),
  ),
);
```

コントローラーが0.0→1.0と動く中で、fadeは最初の50%、slideは30%〜100%の区間でそれぞれ動く。1つのコントローラーで複雑な振り付けが実現できる。

---

## まとめ

Flutterのアニメーションシステムは、「時間管理（AnimationController）」「値変換（Tween/Curve）」「UI反映（AnimatedBuilder）」という3層構造で成り立っている。この構造を理解すると、複雑なアニメーションも「どのレイヤーで何をするか」に分解できる。

暗黙的アニメーションはシンプルなケースに強く、明示的アニメーションは制御が必要なケースに強い。Heroアニメーションは画面遷移に特化した便利な仕組みだ。

アニメーションはユーザーに「このアプリは丁寧に作られている」と感じさせる重要な要素だ。仕組みを理解した上で、適切なアニメーションを選んで使ってほしい。
